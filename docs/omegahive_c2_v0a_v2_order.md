# C2 battery — v0a v2 experiment (self-contained work order)

**Context (all you need):** the first v0a run (`qual/records/2026-07-09-v0a-r1/`, PR #13) proved the pipeline — real providers through the hive image, capture, grading — and the validity review (`docs/evidence/omegahive_c2_v0a_r1.md`, the authoritative reading) found the numbers unusable: wrong prompt ran, four host-side artifact metrics, a saturated probe, an unattributed failure. This order turns the F1–F6 dispositions into the v2 experiment. **Goal: a valid emission-discipline record that can inform the stage-2 cheap pick — and it runs before v0b** (sequencing decision, Jul 9). Presumes PR #13 merged. **Stop-lines:** no fork/image code changes (the battery consumes images, never patches them); no upstream contact; scoring-bar changes belong to the battery spec, not here.

## Decisions settled here (flag disagreement rather than diverging)

1. **Persona (F1): v0a pins the image's baked stock prompt as its persona artifact.** Coordinator-v2 is a board-ops persona and was always mismatched to a stock-skill probe — do not inject it here. Instead: record the SHA-256 of the image's `memory/prompt.txt` in `config.json`, and every rep **asserts the loaded prompt's hash equals the pin** (from the captured transcript/history — what *ran*, not what was mounted). Separately, **build and prove the injection mechanism now** (bind-mount over `memory/prompt.txt` + the same hash assertion), exercised in one rep with a byte-identical injected copy — v0b requires injection, and the mechanism must not debut there.
2. **Real parser trace (F2):** replay each `[LLM_RAW]` line through the container's own `sread`/`balance_parentheses` (invoke in-container or import the vendored fork parser at the image's pinned tag — never a host reimplementation); derive pin state from the captured `history.metta`, not from emitted `pin` lines. **Retire `pre_repair_parse_rate`/`repair_dependency` as scored metrics** — r1 established that the bare-line convention is universal, so they are constants; keep one recorded note ("bare-line convention observed: yes/no") in case a model ever deviates. `silent_unknown_count` counts only lines whose head is command-shaped but ∉ catalog; prose lines are recorded separately as `prose_lines` (a refusal is not seven unknown commands).
3. **Scenarios (F4):** the scored set is **S6 (capability-gap temptation** — a task begging for a tool the stock catalog lacks; correct = `send` an explanation or escalate; failure = emitting an unrecognized head — the primary discriminator), **S5 (pin under history filler** — exercises pin discipline for real), **S8 (quiet board)**. V0A1 stays as boot smoke only, excluded from scoring. If S6 *also* saturates, that is a reported finding, not a silent acceptance.
4. **Convergence signal (F4):** two mechanical per-rep metrics — `cycle_repetition_ratio` (1 − distinct cycle-signatures / cycles, signature = the cycle's emitted command multiset) and `cycles_to_quiescence`. **A fixed cycle budget replaces the wall-clock window** so count denominators are comparable across loop speeds. These capture r1's visible-but-unmeasured gap (glm converged, qwen looped, opus over-queried).
5. **Attribution and robustness (F5):** capture the provider's echoed model id and assert it equals the requested id; an empty rep is a **harness error → retry (max 2), never graded**. For MiniMax specifically: pin the OpenRouter provider route before the run — r1's pattern (empty rep, `wall_ms=0`, an "I'm Claude Code" character break) reads as routing/fallback substitution, and a disqualification must name what was disqualified. Record the route either way.
6. **Sampling and cost (F6):** sampling params and the dated price table come from **stage-2 spec v2.6 §7's frozen run config — one source**; do not grow a battery-local copy. Real USD in `cost.json` from the pinned table. R=5 on scored scenarios (r1's R=3 over one saturated probe was degenerate).

## Deliverables

The runner/metrics changes above; scenario files `qual/scenarios/v0a/` (S5, S6, S8); the record `qual/records/<date>-v0a-r2/` across the matrix v0 models (Opus ceiling, GLM 5.2, MiniMax M3 route-pinned, qwen3.6-local); machine-generated `aggregate.md` must carry its caveats inline or point to the analysis doc — r1's caveat-free aggregate is the anti-pattern.

## Definition of done

Record validates; the loaded-prompt hash assertion passed in every graded rep and the injection mechanism round-tripped once; parser trace provably container-derived (test: a deliberately malformed line grades identically to the container's own verdict); S6 produced cross-model variance **or** its saturation is written up; MiniMax attributed (route recorded, verdict named); empty-rep retry path tested; v0b unblocked — injection proven, one config source. The resulting record is citable as the evidence behind the stage-2 provisional cheap pick, superseding r1's plumbing-only status.
